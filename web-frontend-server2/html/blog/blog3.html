🔀 Bài 3: Nginx Reverse Proxy & Load Balancer

NginxProxyLoad Balancer

📅 10/01/2025 · Tran Van Huy

🔀

1. Reverse Proxy là gì?

Reverse Proxy là server trung gian nhận request từ client và chuyển tiếp đến backend server phù hợp. Client không biết server thật đứng sau là gì – giúp ẩn cấu trúc nội bộ và tập trung SSL.

2. Cấu hình Nginx trong MyMiniCloud

upstream web_backends {
  server web-frontend-server:80;   # Server 1
  server web-frontend-server2:80;  # Server 2 (Round Robin)
}
server {
  listen 80;
  location /       { proxy_pass http://web_backends; }
  location /api/   { proxy_pass http://application-backend-server:8081/; }
  location /auth/  { proxy_pass http://authentication-identity-server:8080/; }
  location /student/ { proxy_pass http://application-backend-server:8081/student; }
}

3. Round Robin Load Balancing

Với 2 web server, Nginx phân phối request luân phiên: Request 1 → Server 1, Request 2 → Server 2, Request 3 → Server 1, ... Giúp tăng khả năng chịu tải và tính sẵn sàng cao (High Availability).

4. Lợi ích thực tế

  • Một điểm vào duy nhất (port 80) cho toàn hệ thống
  • Tự động phân tải giữa nhiều backend
  • Dễ dàng mở rộng thêm server mà không đổi URL
  • Bảo vệ backend khỏi expose trực tiếp ra internet
